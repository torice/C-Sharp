Why C#?

Motivation

- Java lacked cross-language interoperability (mixed-language programming): ability for code produced by one language to work easily with code produced by another

--problem for distributed software systems

--problem for software components

- Windows by far the most widely used OS

- Yet Java lacked direct support for Windows; could not fully integrate with Windows platform

Design Goals

C# is intended to be a simple, modern, general-purpose, object-oriented programming language.

- For developing software components deployable in distributed environments

  • Develper Support:
    • - Software robustness, durability
    • - Programmer Productivity
    • - Source Code Portability
    • - Progammer Portability (C and C++ history)
    • - Internationalization

  • Coding Support:
    • - Strong Type Checking
    • - Array Bounds Checking
    • - Auto Garbage Collection
    • - Uninitialized Variable Checks

  • Performance:
    • - C# aims to be economical with memory and processing power
    • - But not intended to perform on level of C or assembly

What makes C# stand out?

- Built from the ground up to be an enhanced object-oriented programming language

- Fixed certain shortcomings of Java


  • .NET Framework
    • - C# written to work in tandem with .NET
    • - Language Interoperability
    • - Large Class Library (FCL)
      • -- A set of ready-made classes that simplify a lot of the developers most common work
      • -- Database connections, network and disk access, security and authentication
    • - Execution in CLR virtual machine

    • - Large community support

    - Build all sorts of apps

    - Internationally standardized

    - Direct support for Component-Oriented programming

    - Built-in functional programming capabilities

    - Built-in asynchronous programming capabilities

    - Native garbage collection

    What can C# do?

  • - You can build nearly any kind of application using C# and related tools.
    • - Windows clients applications
    • - Web apps
    • - Mobile apps
    • - Enterprise software
    • - Backend and service-oriented applications

    • Advantages over Java and C++

    - Properties: formalized concept of get-set methods, so the code becomes more legible

    - Events management: using delegates

    - CIL (Common (.NET) Intermediate Language) is a standard language, while java bytecodes aren't

    - Has more primitive types (value types), including unsigned numeric types

    - Indexers let you access objects as if they were arrays

    - Conditional compilation

    - Simplified multithreading
