Comparison

What c# has "fixed"

  • Can switch on strings.
  • For loop scoping: 
    for(int i = 0; i < 3; i++) {
    ;
}

// In c++, 'i' is still accessible at this point.
// c# decided to fix this.

What C# has "broken"

  • For loop scoping: 
    // Will compile in c++ but not c#.
for (int i = 0; i < 3; i++) {
   int j = 3;
}

int j = 5;

// C# compiler complains that 'j' has already been defined in the second 'j assignment'.

Conventions

  • Method names use unique variant of camel case, sometimes referred to as UpperCamelCase. 
    // C# method call.
object.MethodCall();

// Java method call.
object.methodCall();

Variable Declaration

  • C# is both strongly and weakly typed.
  • Variables may be explicitly typed: 
    string sentence = "Hey dare.";
  • Variables may also be implicitly typed: 
    var sentence = "Hey dare.";
  • C# conventions suggest the use of 'var' type only when type of object is obvious from the right hand side of assignment. 
    // Recommended.
var sentence = "This is clearly a string.";
var number = 27;

int var4 = Example.ThisCouldReturnAnything();

Writing to Console

  • // C#
Console.WriteLine("String to write");

  • // Java
System.out.println("String to write");

The "this" Object

  • Similar syntax to Java.
  • "this" object/pointer is not available in static methods. 
  • // Example usage of "this".
this.MethodCall();

Runtime environment

  • While Java has JVM, C# runs on top of the .NET "common language runtime".
  • Both allow code to be managed at execution time.
  • Executables are portable due to .NET runtime layer.

Performance

  • Java and C# are pretty even when it comes to performance.
  • In certain environments C# will excel specifically due to more efficient implementation of generics.
