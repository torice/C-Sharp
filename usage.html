Our goal is to give you an idea of why C# exists and the driving force behind why it was created. We also want to show you some of the secrets of C# and how you can take advantage of them.

What we've learned

Today we are here to share what we have learned about C#, why J++ no longer exists, and how C# is now Microsoft's magic script for keeping the .net dream alive. We'll show you some of it's tools, why it was created, and some examples of some C# code.

Uses for C#

Since C# is a managed language, it ensures there aren't any memory leaks. In simple programs, this ensures the performance of the system the application is running on is optimized. For high end programs and applications, this can have a negative impact. For instance, games like Doom 3 and 4 would not work well if the code was written in C#. Enabling unsafe mode can improve calls between managed code. One scenario that requires unsafe mode is in the situation where a developer needs to use pointers.

Uses for C#

If we take a look at C# from the "30 thousand foot view" we can see that it borrows many concepts from Java, C, and C++. This makes the code look very familiar to most programmers but it was designed to be easier to use with syntax that allows the code itself to basically explain what it is doing. With a built in library containing many preconfigured functions, writing clean, readable, and portable code becomes much easier. With these features in mind and the fact that it is an object oriented language, it makes for an excellent language that can be used in rapid application deployment.
